# IDS 系统性能优化指南

## 1. 数据库优化

### 连接池配置

```bash
# 生产环境推荐配置
IDS_DB_POOL_SIZE=20
IDS_DB_MAX_OVERFLOW=40
IDS_DB_POOL_TIMEOUT=60
IDS_DB_POOL_RECYCLE=1800
```

### 索引优化

确保以下字段有适当的索引：

- `alerts.timestamp`
- `rules.category`
- `rules.enabled`
- `rules.priority`

## 2. 内存优化

### 规则引擎优化

- 规则按优先级排序，优先匹配高优先级规则
- 使用多级索引：协议 → 端口 → IP
- 定期清理过期上下文（默认5分钟）

### 缓冲区管理

```bash
# 调整缓冲区大小（字节）
IDS_MAX_BUFFER_SIZE=524288  # 512KB
IDS_CONTEXT_TIMEOUT=600     # 10分钟
```

## 3. 网络优化

### 网络接口选择

```bash
# 生产环境使用实际网卡
IDS_NETWORK_INTERFACE=eth0

# 避免使用lo（回环），除非测试
# IDS_NETWORK_INTERFACE=lo
```

### 数据包过滤

- 只处理IP数据包：`filter="ip"`
- 避免存储完整数据包：`store=0`

## 4. 并发优化

### 异步处理

- 使用asyncio处理WebSocket广播
- 数据库操作使用连接池
- 规则匹配在单独线程中进行

### 线程管理

- 嗅探线程设为守护线程
- 规则重建使用后台线程
- 避免线程竞争，使用RLock

## 5. 监控和调优

### 性能指标

- 监控PPS（每秒数据包数）
- 跟踪匹配成功率
- 观察内存使用情况

### 日志优化

```bash
# 生产环境使用适当的日志级别
IDS_LOG_LEVEL=WARNING
IDS_LOG_TO_FILE=true
```

## 6. 部署建议

### 系统配置

```bash
# 增加系统缓冲区
sysctl -w net.core.rmem_max=262144
sysctl -w net.core.wmem_max=262144

# 优化网络栈
sysctl -w net.ipv4.tcp_rmem="4096 87380 262144"
sysctl -w net.ipv4.tcp_wmem="4096 87380 262144"
```

### Docker部署

```yaml
# docker-compose.yml
services:
  ids:
    image: ids:latest
    environment:
      - IDS_DB_POOL_SIZE=30
      - IDS_DB_MAX_OVERFLOW=60
    cap_add:
      - NET_RAW
      - NET_ADMIN
    networks:
      - host
```

## 7. 故障排查

### 常见性能问题

1. **高CPU使用率**：检查规则数量，考虑规则分组
2. **内存泄漏**：监控ContextManager和FlowReassembler
3. **数据库连接超时**：调整连接池参数
4. **WebSocket延迟**：检查广播队列积压

### 调试命令

```bash
# 监控系统资源
top -p $(pgrep -f "python.*main")

# 查看网络流量
iftop -i eth0

# 数据库连接状态
psql -c "SELECT count(*) FROM pg_stat_activity WHERE datname='ids_db';"
```
